# 公众号爆款写作技能

## ⚠️ 写作第一原则：内容深度 > 朱雀检测

**公众号是商业变现链条中最关键的一环。目标不是过朱雀，是写出有深度、有价值、对读者有用的内容。**

- 标题决定点击率
- 开头决定完读率
- 内容深度决定转化率
- 朱雀检测是最后一步的保险，不是写作的驱动引擎

**写作时按这个顺序思考：**
1. 这篇文章的核心价值是什么？读者看完能带走什么？
2. 标题能不能让人想点进来？
3. 开头能不能把人留住？
4. 框架逻辑是否清晰？每一段是不是在推进一件事？
5. 内容颗粒度够不够？数字够不够具体？案例够不够真实？
6. 结尾有没有互动引导？
7. 最后一步：朱雀检测自检

如果内容质量够好，即使朱雀检测不是100%人工，照样可以发。朱雀检测是辅助工具，不是裁判。

## ⚠️ 执行规则（必须严格遵守）

### 规则1：用户指令优先级最高

**用户给了限定条件 → 必须严格执行，不准自己发挥。**

| 用户指令类型 | 你的做法 |
|-------------|---------|
| 给了参考爆文 + 指定观点 + 指定素材 | 高度仿写参考爆文，观点/素材严格按用户要求，不准自己加戏 |
| 只给了观点，没给参考文 | 可以按方法论自由发挥 |
| 指定了视角（如"用孙宇晨案例"） | 全文必须围绕指定素材，不准乱换 |
| 指定了人称（如"用第三人称"） | 全文必须统一人称，不准中途切换 |

**❌ 禁止行为：**
- 用户没让你写"我"的故事，你私自编造"我"的经历安在自己身上
- 用户指定了素材（如孙宇晨），你中途又加了一堆"我"的内容
- 用户给了参考爆文，你不按参考文的结构/观点写，自己另起炉灶

**不确定时，先问用户，再动笔。别写一堆垃圾出来，更浪费时间。**

### 规则2：高度仿写时，必须完全跟随参考爆文

当用户提供参考爆文时：
1. 先按三段式拆解参考爆文（第0步）
2. 拆解完成后，观点/结构/语气/节奏，全部严格跟随参考爆文
3. 指定的素材（如孙宇晨案例），完整融入参考爆文的结构中
4. 不准中途改变视角、不准加参考文里没有的内容

### 规则4：从全书找素材，不只用用户划线部分

**凡是用户让你找某本书里的素材，除非用户明确指定章节，否则必须从全书所有内容里找最适合主题的。**

| 错误做法 | 正确做法 |
|---------|---------|
| 只用用户划线/标注的部分 | 读全书目录，遍历所有章节，找最贴合主题的素材 |
| 用户没指定章节，就随便找一个 | 先浏览全书章节目录，找到最贴合主题的3-5个章节，再从中选最合适的一个 |
| 书里没有明确写，就自己编 | 如实告诉用户"书里没有写这个"，请用户确认或更换素材 |

**操作步骤：**
1. 获取书籍全文或完整目录（190+章节必须全部过一遍）
2. 根据文章主题，筛选最贴合的章节
3. 读筛选出的章节详细内容，提取真实素材
4. 如果书里确实没有合适素材，如实告知用户，不自己编

---

### 规则3：人称和视角必须全文统一

| 错误做法 | 正确做法 |
|---------|---------|
| 开头第一人称"我"，中间突然变成第三人称 | 全文统一人称，要么全"我"，要么全"他" |
| 参考文没有"我"的故事，你私自加"我"的经历 | 参考文用什么视角，你就用什么视角 |
| 一会用孙宇晨案例，一会又跳去讲"我" | 素材统一，视角统一，逻辑一以贯之 |

## 写作前的认知准备

### 变现前置思维

**变现不是做完内容才想的事，是起号之前就要规划好的。**

- 10倍比2倍更容易：与其努力提升2倍，不如换一条能10倍的路径
- 商业模式对标：找到已验证的变现模式，对标着做
- 写每篇文章时想清楚：这篇文章服务于变现链条的哪个环节？

### 定位三要素

1. **定身份**：你是谁？凭什么说？（Vicky：44岁，AI+自媒体实战派，全网15w粉）
2. **定价值**：你能帮读者解决什么问题？（赚钱方法+成长认知）
3. **定赛道**：你在哪个领域深耕？（AI赚钱/自媒体变现）

### 定位三原则

1. **离钱近**：选离交易近的赛道，变现路径短
2. **做垂直**：越垂直，越容易建立专业度和信任
3. **取所长**：用你的存量资源和个人优势，不要从零开始

### 五维决策框架

选赛道/选题时，用5个维度评估：
1. **短期可成**：3个月内能不能出结果？
2. **存量资源**：你手头有什么可以用的？
3. **熟悉圈层**：你的用户在哪些圈层？
4. **形象特质**：你的个人特质和赛道是否匹配？
5. **商业回报**：这个赛道的变现天花板有多高？

> 详细的变现/涨粉/运营方法论见 `references/business_strategy.md`

## 写作全流程

按以下7步顺序执行，每一步完成后才能进入下一步：

### 第0步：拆解参考爆文（用户提供了参考爆文时必须执行）

**当用户提供参考爆文/爆款资料时，必须先用以下标准三段式拆解参考文章，然后基于拆解结论来仿写。不要跳过这步直接写。**

#### 拆解标准结构（三段式）

**第一部分：基础信息**
- **账号来源**：定位、粉丝量、领域
- **字数拆解**：总字数 + 各部分（开头/01/02/03/结尾）字数分布
- **数据表现**：本文数据（阅读量/点赞/分享/完读率/推荐占比）vs 账号日常数据的倍数关系。关注打开率、分享转化率等关键指标。如有延迟推荐等特殊现象需标注

**第二部分：内容拆解（10个维度逐项分析）**
- **01 选题**：选题类型、来源、不可替代性/壁垒、冒犯程度（如有）、与目标读者的匹配度
- **02 标题**：技巧拆解（对应8大技巧）、结构分析（可提炼为模板）、关键词分类（数字/痛点/共鸣/热点/圈层/故事）
- **03 开头**：写法概括（哪种开头技巧）、技巧拆解（逐句分析为什么有效）
- **04 大框架**：框架类型（干货三件套/观点三步法/并列式/递进式等）、与小标题的对应关系
- **05 细脉络**：逐段拆解，每段标注功能（引入/论证/案例/总结/落点），用表格呈现
- **06 表达手法**：整体手法（类比/并列/对比/具象化/名人引用等）+ 局部亮点（具体词汇、句式）
- **07 结尾**：技巧分析（金句收尾/总结升华/制造共鸣等）、情绪收束方式
- **08 爆款成因**：至少分析4-6条成因（冒犯性/身份效应/选题匹配度/算法机制/标题悬念/篇幅节奏等）
- **09 金句**：划线前3金句逐句分析手法和可迁移性
- **10 其他亮点**：文中独特的技巧、概念体系、情绪曲线设计等

**第三部分：迁移借鉴**
- **01 选题迁移**：同类选题 + 跨领域选题
- **02 标题迁移**：提炼可复用标题框架（至少3-4套），每套给出套用示例
- **03 框架迁移**：本文框架 + 可迁移场景
- **04 开头结尾迁移**：提供可套用的写法范例模板
- **05 同类爆文推荐**：相关选题方向的文章推荐

**可选第四部分：避坑指南**
- 文章的可优化点，避免踩同样的坑

#### 拆解后仿写流程
拆解完成后，基于拆解结论来搭框架、拟标题、写正文。拆解中提取的标题框架、开头写法、结尾金句、表达手法，都要在新文章中有所运用。目标是"站在爆文的肩膀上写"，而不是凭空创作。

> 详细的拆解案例见 `references/` 目录下的爆文拆解文件，共40+篇拆解案例。
>
> **100%人工创作文风指南**见 `references/writing_rules.md` 第24节，基于25篇100%朱雀人工认证爆文分析得出，用于确保输出文章无AI味。

### 第1步：判断文章类型

收到写作任务后，先判断文章属于哪种类型，不同类型的写法完全不同：

| 类型 | 判断标准 | 开头方式 | 代表作 |
|------|---------|---------|--------|
| 观点文 | 表达一个反常识/颠覆认知的观点 | 直抛核心观点，1-2句说完 | 《如果你的存款有二三十万，请牢牢记住这3条保富法则》 |
| 干货文 | 交付一套可操作的方法/步骤 | 用痛点/问题引入 | 《一个很笨，但能让你变有钱的方法》 |
| 案例文 | 用真实人物故事说明一个道理 | 人物背景+事件引入，场景代入 | 《前大厂产品经理裸辞，做AI博主，单价从900涨到6500》 |
| IP长文 | 建立个人品牌，让读者认识你 | "我为什么写这篇文章" | 自我介绍文/转折故事文/里程碑复盘文 |
| 销售软文 | 推广产品/课程/社群，同时有内容价值 | 讲用户渴望的结果/代入感强的改变故事 | 课程推广文/社群招募文 |
| 深度学习笔记 | 深度学习某个人/某本书/某门课后输出 | "我深度学习了XX，对XX有了新的理解" | DanKoe深度学习系列12篇 |
| 读书观点文 | 重读经典后提取认知/方法 | "40岁重读XX才发现+反常识观点" | 《天道》系列、《肖申克》系列 |
| 访谈金句清单体 | 提炼访谈/演讲的金句精华 | "X小时/X万字...提炼出X条真相" | 罗永浩/段永平/刘震云/章小蕙系列 |
| 成长分享文 | 分享个人成长经历和感悟 | 轻松打招呼+场景引入 | 《82年，44岁，副业月入3w》系列 |

**关键区别**：
- 案例文开头是"讲故事"——谁、什么处境、发生了什么
- 观点文/干货文开头是"抛观点"——直接给核心认知，不绕弯
- IP长文开头是"交底"——为什么写、我是谁、发生了什么变化
- 销售软文开头是"勾欲望"——先让读者看到可能性，再递方案
- 深度学习笔记开头是"交学习过程"——我学了什么、怎么学的、发现了什么
- 读书观点文开头是"重读+颠覆"——重新读后发现以前没看到的东西
- 访谈金句清单体开头是"信息量+我的付出"——X小时、X万字、我看了X遍
- 成长分享文开头是"轻松聊天"——像发朋友圈一样开头

### 第2步：拟定标题

**标题8大技巧**（来自25讲第08讲，57%的爆文标题含数字）：

1. **激发好奇**：制造信息缺口，让人想点进去看答案
2. **激发认同**：替读者说出他们心里的话
3. **展示负面**：警示风险，让人不得不看
4. **展示回报**：告诉读者看完能得到什么
5. **展示新闻**：借热点/新事件的时效性
6. **呈现对比**：用反差制造冲突感
7. **颠覆认知**：打破常识，引发好奇
8. **事实+观点**：前半句说事实，后半句给观点（案例文标配）

**标题硬性规则**：
- 字数：15~30字（平均25字）
- 必须含有具体数字
- 可用感叹号（30%），少用问号（13%）
- 算法敏感词规避：冲推荐的文章，标题避开"搞钱/赚钱/副业"等词，用"破局/跃迁/思维方式/认知"替代
- 商稿标题风格：数字+反常识，优先保粉丝打开率
- 一次给出3个标题备选，让用户选择

**标题结构增补（来自孙宇晨文章验证）**：

**结构：时间对比+双事实+观点**（适用于观点文/案例文，尤其适合有多年跨度的人物故事）

```
[时间1]+[事实1]，[时间2]+[事实2]，[人物名]+[观点/方法]：核心认知
```

示例：
- `十年前劝90后千万别买房，今天一句话让行业暴涨487%，孙宇晨赚钱独门心法：看透人性`
- `罗翔3200万粉丝之后，我才明白：真正厉害的人，都有这个特质`

写法要点：
- 前半句用两个时间点/事实制造跨度，都有具体数字
- 中间出现人物全名（不用"他"），增强辨识度
- 冒号后直接给观点/方法，4-8个字，不加解释
- ❌ 禁止全文用"他"开头导致主语模糊
- ❌ 禁止冒号后加长句解释

**结构：事实+观点**（案例文标配，来自25讲第08讲）
- 前半句说事实（含数字），后半句给观点
- 示例：`前大厂产品经理裸辞，做AI博主，单价从900涨到6500`

### 第3步：搭建框架

**8种爆文框架**（来自25讲第09讲）：

| 框架 | 适用类型 | 核心结构 | 示例 |
|------|---------|---------|------|
| 干货文框架 | 干货文 | 是什么→为什么→怎么做 | 教方法的干货文 |
| 观点文框架 | 观点文 | 总→分→总（抛观点→分论点→收束） | 保富法则、花钱观点文 |
| 清单体框架 | 干货文/观点文 | N个要点并列呈现 | "3条法则""5个方法" |
| 裂变法框架 | 干货文 | 痛点→方法→效果→更深层方法→更大效果 | 递进式交付 |
| 人物稿框架 | 案例文 | 发现机会→怎么赚钱→普通人怎么学 | 栗噔噔案例文 |
| 同维度并列 | 观点文 | A/B/C并列论证同一观点 | 多个反面案例并列 |
| 逻辑递进 | 观点文/干货文 | A→B→C层层推进，后一个基于前一个 | 递进论证 |
| 不同维度并列 | 观点文 | 从不同角度论证同一观点 | 心理+行为+结果 |

**案例文专用框架**（Vicky手改终稿验证）：
```
【标题】前事实+后观点
【开头】人物背景→关键事件→抛出钩子（为什么？怎么做到的？）
【01】她是怎么发现机会的——从什么点切入的
【02】她现在怎么赚钱的——具体的赚钱路径和故事
【03】普通人怎么学——拆解出可复制的方法
【结尾】呼应主题+共勉+互动提问
```

**IP长文专用框架**（来自25讲第18讲）：
```
【标题】个人故事+转折/认知
【开头】我为什么写这篇文章
【01】我是谁——背景、经历、起号故事
【02】我在做什么——现在在做什么、为什么做
【03】为什么做——价值观、长期目标
【结尾】感谢+邀请同行
```

**销售软文专用框架**（来自25讲第23讲）：
```
【标题】渴望结果/改变故事/痛点
【开头】讲用户渴望的结果 或 代入感强的改变故事
【01】大痛点——为什么这个问题必须解决
【02】解决方案——你提供什么（自然引出产品）
【03】效果证明——案例/数据/学员反馈
【结尾】行动呼吁+限时/限量（如适用）
```

**深度学习笔记专用框架**（来自12篇DanKoe系列）：
```
【标题】40岁深度学习XX才发现+核心认知/方法
【开头】"我深度学习了XX，对XX有了新的理解"→用2-3个金句预告→强调作者付出（X天/X遍）
【01】是什么——核心概念/认知是什么
【02】为什么——为什么这个认知重要/颠覆
【03】怎么做——普通人怎么用（实操步骤+自身实践）
【04】写在结尾——DK/XX说的核心金句+你我共勉
```

**读书观点文专用框架**（来自5篇《天道》/《肖申克》系列）：
```
【标题】[年龄]重读/三刷[书名]才发现+反常识观点
【开头】场景引入（1v1咨询/普遍痛点）→建立权威性（读了X遍/做了X个视频）→抛核心观点→提出疑问引导
【01】书中案例详细拆解+现实对照（对比：成功vs失败角色）
【02】方法论提炼+自身经历印证
【03】（可选）升华主题+金句总结
【结尾】总结核心观点+金句+"你也可以"+互动引导
```

**访谈金句清单体专用框架**（来自9篇访谈系列）：
```
【标题】X小时访谈，X万字文稿，XX对谈XX，才发现，X个反常识真相（值得收藏）
【开头】热点引入→金句快速预告（3-5句最冲击的原话）→"我"的付出（看了X遍/用了X天）→价值承诺
【01-X】分板块罗列金句，每条=标题句+1-3句展开解释
【结尾】排比总结+引用原话金句+你我共勉+互动引导
```

**第三人称观点文专用框架**（来自孙宇晨文章验证，适用于写他人故事来表达观点的文章）：
```
【标题】[人物名]+[核心事实/观点]（人物全名直接出现在标题）
【开头】热点切入（第一句：最近发生了什么）→具体事实+数字→时间对比（过去vs现在）→引出核心观点
【01】关键转折事件详细叙述（事件链驱动，有场景/原话/冲突）
【02】看透人性/规律后怎么做事（多个案例佐证）
【03】普通人怎么用（落到读者身上，给方法/认知）
【结尾】总结核心观点+金句+"你我共勉"+互动引导
```
写法要点：
- 开头❌禁止第一人称"我"的故事切入（本文用第三人称，不代表所有文章都禁止"我"，按文章类型决定）
- 人物全名在标题和开头都要出现，不用"他"
- 01章节必须是事件链驱动，不是性格描述
- 适用场景：写名人/他人故事来表达一个观点的文章
**清单体两种形态**：
- 金句型：编号列表，每条50-150字，总量20-30条，碎片化友好
- 解读型：3个核心观点深度展开，每个800-1500字，有案例有理论

**成长分享文专用框架**（来自20篇IP+成长系列）：
```
【标题】[年龄标签]+[个人成绩]+才发现+反常识感悟
【开头】轻松打招呼+场景引入（今天遇到了XX/有小伙伴问我XX）
【01】个人经历故事（从A到B的转折过程）
【02】方法/感悟/认知提炼
【03】（可选）产品/课程自然植入
【结尾】总结+你我共勉+互动引导+（可选）产品引流
```

**框架搭建输出**：列出完整的01/02/03小标题，让用户确认后再写正文。

### 第4步：写正文

写正文时，遵守以下全部规则。详细规则和示例见 `references/writing_rules.md`。

**核心铁律（最重要——做不到这一条其他免谈）**：

**🔥 在讲一件连续的事，不是在描述一个人。**

| ❌ AI写法（我犯过的错） | ✅ Vicky写法（你的方式） |
|------------------------|------------------------|
| "她管自己叫'一人公司'。一个人，加一个AI舰队。"（碎句表演，没有信息） | "她现在管自己叫'一人AI战舰'。一个人决策，多个AI助理执行。"（完整、有信息、在推进） |
| "说个有意思的事儿。"（废话铺垫） | 直接说事，不铺垫 |
| "到现在呢？一篇文章6500。视频5500。"（断句表演） | "从第一个商单900块，到现在的6500块，不到一年，单价飙了7倍多。"（完整句子，一条信息） |
| 一段里塞5个数字（堆砌感） | 数字跟着事件散落在不同段落 |
| 每句话短到2-3个字（电报式） | 正常说话的长度，一句话说清楚一件事 |

**具体写作规则**：

1. **句子完整，逻辑清晰**：正常人说完整句子。不是短才叫口语化。一句话说清楚一件事。信息充分，逻辑链完整。

2. **对话感**："你""我"全程在场（30篇100%使用"你"，97%使用"我"）。用"对大多数普通人而言"替代抽象的"人们""大家"。像在对一个朋友讲这件事。

3. **真实感**：用真实人名+真实数字+真实决策细节。97%的文章用数据/数字支撑观点。拒绝空泛道理，每个观点必须有案例佐证。只写素材里实际发生的事，不杜撰。

4. **节奏感**：长短句交替。不需要刻意把所有句子砍碎。段落间空行分隔。金句独立成段。

5. **行动感**：多用动词，少用形容词堆砌。给确定性建议（"一定要""绝对不要"），不用"可能""也许""我觉得"等模糊词。

6. **字数控制**：按文章类型定字数，不灌水：
   - 观点文/干货文：1500~2500字
   - 案例文：2000~3000字
   - 深度学习笔记：2000~3000字
   - 读书观点文：2000~3000字（内容密度高可以更长）
   - 访谈金句清单体：2500~4000字（金句型可以更长）
   - 成长分享文：1500~2500字

**案例文/第三人称观点文特有规则**（来自孙宇晨文章逐字改稿验证）：

**案例写法核心：事件链驱动，不是性格描述**

| ❌ AI写法（我犯过的错） | ✅ Vicky写法（你的方式） |
|----------|----------|
| 每段在刻画性格特征（"她很节俭""她从小就不花钱"） | 每段都在推进一个**具体动作/事件** |
| 信息一开始全盘托出（人物素描式） | 信息逐步暴露，读者跟着"发现" |
| 像在写人物介绍/议论文/性格分析 | 像在跟朋友讲一件事的经过 |
| 结构：背景→性格描述→事件→感悟 | 结构：事件→事件→事件→事件→收束 |

**事件链写法具体要求**：
1. **有具体场景**：不是"他决定做波场"，而是"身边的人劝他再等等。等政策明朗，等风向稳定，等别人先探路。他没等。"
2. **有原话/直接引语**：把人物说过的话直接摆出来，不是转述大意
3. **有冲突/张力**：不是平铺直叙，而是"所有人都在犹豫观望 vs 他果断出手"
4. **观点在故事讲完之后自然引出**：不是先说观点再举例，而是讲完故事→"那些劝他等的人，输在哪？"→引出观点
5. **"我"只在关键处出现**（适用于第一人称文章）；第三人称文章全程不用"我"
6. **❌ 禁止编造书中没有的素材**——所有案例细节必须从书里找，书里没有就如实告知用户
7. **❌ 禁止贴标签**——不用"他是个果断的人"，而是用"他没等"让读者自己得出结论

**表达风格补充**（来自逐字改稿过程中纠正的问题）：
- ❌ 禁止过于书面的词汇（如"达摩克利斯之剑"→改用直白表达；"一波三折"→偏书面；"理所应当"→"觉得是应该的"）
- ❌ 禁止绕口的总结句（如"那是透析了趋势与人性之后的笃定"→改直给："那是看懂了趋势之后，该出手就出手。"）
- ❌ 禁止结尾排比（三个"谁"字句→删掉不影响意思就不要）
- ✅ 金句收束要短，4-8个字最好，直接给观点，不加解释

**小标题写法增补（来自孙宇晨文章验证）**：

**风格要求**：
- 短，一句话甚至几个字
- 带好奇心或冲突感，让人想往下看
- 格式：`## 01、标题？（带标点）`——数字后加顿号，标题后加问号或句号
- 可以是疑问句（"他凭什么全身而退？""457万买一顿饭，图什么？"）
- 可以是行为描述（"看不起模仿""习惯性等靠要"）
- 可以是短概念（"破情关""破心贼"）

**❌ 小标题禁止句型**：
- 禁止"不是…而是…"（如"赌的是时间，不是运气"❌）
- 禁止过于抽象的总结式小标题（没有好奇心钩子）
- 禁止太长（超过15字）

**自检方法**：写完小标题，问自己——读者看到这个小标题，想不想继续往下看？如果答案是"还行"，就是失败的。

**各类型写作特有规则**：

**深度学习笔记**：
- 连续系列用"这是我深度学习XX的第X篇笔记"建立追踪感
- 每篇聚焦1个核心认知，不贪多
- 必须有"我是怎么1:1复刻/实践的"段落——让方法可落地
- 前后篇呼应："上一篇我们聊了XX，今天来聊聊XX"

**读书观点文**：
- 不是写书评，是从书中提取一个和赚钱/成长相关的认知
- 书中案例要详细叙述，像讲故事，不要一笔带过
- 必须"书中案例+作者亲身经历+现实案例"三重印证
- 每个小节至少有1个金句
- 金句来源：书中台词改编+原创+名人名言

**访谈金句清单体**：
- 开头强调"我"的付出（X天/X遍/X万字），替读者省时间
- 金句型：每条独立可消费，编号从01开始，拉到哪读哪
- 解读型：3个核心观点深度展开，有理论有实操
- 同一素材可以产出2-3篇不同角度（金句版+解读版+对比版）
- 每个板块小标题要有悬念感

**成长分享文**：
- 开头轻松，像发朋友圈一样自然
- 强调"我也是普通人"降低距离感
- 个人数据背书要反复出现（15w粉/2000+学员）
- 产品植入4种方式：经验分享式/问题解决式/工具推荐式/社群训练营式
- 植入原则：先给价值，再推荐产品，70%价值+30%产品

**遣词造句规则**（详见 `references/writing_rules.md`）：
- 连接词串逻辑："其实"→"但是"→"所以"→"因为"，逻辑链清晰
- 禁止书面语："综上所述""值得注意的是""基于此""由此可见"
- 禁止AI味句式："不是……而是……""真正的……是/不是……"
- 禁止排比/对仗
- 禁止过渡废话："说个有意思的事儿""按说""说白了"——直接说事

**开头5技巧**（来自25讲第11讲）：

| 技巧 | 说明 | 适用类型 |
|------|------|---------|
| 提问式 | 用问题勾住读者，制造悬念 | 观点文/干货文 |
| 反常识事实/观点 | 打破读者预期，引发好奇 | 观点文 |
| 猎奇故事 | 用一个奇特的人/事开头 | 案例文 |
| 新闻热点 | 借热点引入话题 | 所有类型 |
| 对我有用 | 直接告诉读者"这跟你的利益相关" | 干货文/销售软文 |

**开头写法增补（来自孙宇晨文章验证）**：

**热点切入开头**（适用于案例文/第三人称观点文，❌不适用于第一人称文章）：
```
第一句：用最近热点直接开场（"XX又上热搜了"）
第二句：给具体事实+数字（"最近几周，XX暴涨了487%"）
第三句：翻旧账/时间对比（"早在10年前，他就..."）
接着：读者反应（"当时都觉得他在说胡话"）
转折：今天再回头看（数字对比，制造冲击力）
最后：引出核心观点（"不过，XX不是天生的...都源于...让他看透了..."）
```
示例（孙宇晨文章开头）：
> 孙宇晨又上热搜了。
> 最近几周，存储芯片股集体狂飙，尤其是闪迪，被孙宇晨"钦点"之后，6个月的时间暴涨了487%，
> 去年11月他发的推特，也被网友翻出来，他说"短期缺芯片，长期缺能源，永远缺存储"，一早就给投资玩家们指明了方向。
> 当然，这不是他第一次点石成金了。
> 早在10年前，他在公开演讲时，就劝90后【千万别买房】，如果有钱，就去投英伟达、特斯拉、甚至买币。
> 当时所有人都觉得他在说胡话，但今天再回头看，特斯拉涨了49倍，比特币涨了60多倍，英伟达足足涨了201倍。
> 不过，孙宇晨可不是天生的预言家，更没有商业奇才的天赋，
> 他能从小镇青年，一路创办波场...都源于2017年的一次生死抉择，让他彻底看透了，真正能赚大钱的人，只看人性。

**❌ 开头禁止事项（场景化，不是绝对规则）**：
- 第三人称观点文/案例文：❌禁止第一人称"我"的故事切入（本文验证，不代表所有文章）
- 第一人称文章：✅可以用"我"的故事切入
- ❌禁止在开头就描述人物性格（"他是个果断的人"）→ 要讲事件，让读者自己得出结论

**结尾5技巧**（来自25讲第11讲）：

| 技巧 | 说明 | 适用类型 |
|------|------|---------|
| 金句共鸣 | 用一句有力量的话收束 | 观点文 |
| 强化主题 | 再点一次核心观点 | 所有类型 |
| 强化价值 | 强调读者能得到什么 | 干货文 |
| 制造话题 | 抛出一个开放性问题引发讨论 | 所有类型 |
| 呼吁行动 | 引导点赞/转发/评论/购买 | 所有类型 |

> 详细的选题逻辑、素材搜索方法、拆解方法论见 `references/writing_rules.md`

### 第5步：写结尾

结尾自然收束，不强行升华。包含：

1. **回扣主题**：一句话呼应核心观点
2. **共勉**：自然过渡——"你我共勉"或"感谢读完"
3. **互动提问**：1~2个和内容直接相关的问题，格式——"❓️看完XX，你最想……？❓️你手头有没有……？欢迎【评论区】一起聊聊~"

**销售软文结尾**：在互动提问之前，加行动呼吁——"如果你也想……，欢迎……"（自然过渡到产品/课程）

### 第6步：易撰检测（发布前必做）

**易撰检测是文章发布前的硬性门槛，不可跳过。**

检测工具：https://www.yizhuan5.com/

#### 6.1 原创度检测

- ✅ **原创度必须 > 75%**
- ❌ 低于75%必须重写，调整段落结构、更换表达方式、打散相似句式
- 检测维度：与已发布公众号文章、网络内容的相似度比对

#### 6.2 敏感词检测与自动替换

**敏感词会导致平台限流甚至删文，必须在发布前全部规避。**

易撰会把标红的词全部列出，逐一替换为标绿的可用词后才能发布。

**常见敏感词替换对照表**（易撰检测标红后，按此表替换为标绿词）：

| 敏感词（标红） | 替换词（标绿） | 类型 | 说明 |
|---------------|--------------|------|------|
| 特朗普 | 川普 | 政治人物名 | 部分外国政治人物名会标红，逐一检测后更新此表 |
| 其他外国领导人中文名 | 用中文音译变体或通用称呼 | 政治人物名 | 不确定时先用音译变体，检测后再确认 |
| 开枪 / 爆炸 / 暴力相关 | 删掉或换表述 | 恶性行为词 | 涉及暴力、恐怖、极端主义的词，平台会限流甚至删文 |
| 最佳 / 第一 / 最强 / 绝对 / 100% | 领先 / 突出 / 顶尖 / 非常 / 极高概率 | 广告法极限词 | 偶尔用问题不大，通篇滥用会标红 |

**❌ 以下词汇不是敏感词，是用户的账号标签词，系统会根据这些词推荐**：
- 赚钱 / 搞钱 / 副业 —— 用户在19篇10万+爆文中都用了这些词，是内容和账号的核心标签
- 冲推荐的文章，标题避开这些词（算法敏感词），但正文里可以正常使用

**敏感词5大类型**（写作时注意规避）：
1. **政治敏感词**：国家政治、领导人、政策等（如"特朗普"→"川普"）
2. **色情低俗词**：色情、低俗、不雅内容
3. **暴力恐怖词**：暴力、恐怖、极端主义
4. **歧视侮辱词**：种族、性别、宗教歧视
5. **广告法违禁词**：夸张虚假宣传（"最佳""第一""绝对"等）

**写作时的预防措施**：
- 写到外国政治人物时，默认用中文音译变体（如特朗普→川普）
- 冲推荐的文章，标题和正文避开"赚钱/搞钱/副业"等算法敏感词
- 不用广告法极限词（最佳、第一、最强、绝对、100%）
- 写完后通读一遍，把可能触发敏感词的表述提前替换

#### 6.3 检测流程

1. 写完文章 → 自检清单（第7步）全部通过
2. 打开易撰 → 粘贴文章 → 开始检测
3. 查看原创度分数 → <75%则重写
4. 查看敏感词标红列表 → 逐一替换为标绿词
5. 替换后重新检测 → 确认全绿 → 输出最终稿件

> **注意**：易撰的敏感词库会随政策更新，写作时遇到新的标红词，及时添加到本替换对照表中。

### 第7步：自检清单

写完后，按以下顺序逐项检查：

**内容深度（最重要——先过这一关）**：
- [ ] 这篇文章有没有核心价值？读者看完能带走什么认知或方法？
- [ ] 每一段的信息是不是有效的？有没有可以删掉但不影响意思的废话？
- [ ] 案例够不够具体？数字够不够真实？方法够不够可操作？

**结构完整性**：
- [ ] 标题含数字+痛点/反常识？15~30字？
- [ ] 开头有场景代入/抛观点/勾欲望？能让读者"看到"人物或问题？
- [ ] 01/02/03数字分段，每段在推进一件事？
- [ ] 每段有案例/细节佐证？
- [ ] "你""我"全程在场？
- [ ] 结尾有互动引导？

**表达质量**：
- [ ] 句子是完整的、有逻辑的？不是碎片化的？
- [ ] 数字散落在不同段落？没有集中堆砌？
- [ ] 没有排比/对仗/AI味句式？
- [ ] 没有过渡废话（"说个有意思的""说白了"等）？

**小标题质量**：
- [ ] 小标题短小精悍？带好奇心或冲突感？
- [ ] 没有用"不是…而是…"等禁用句型？
- [ ] 能让人想继续往下看？

**排版规范**：
- [ ] 段落短小，手机屏1-3行一段？
- [ ] 重点句加粗？
- [ ] 小标题用01/02/03数字格式？
- [ ] 图片位置合理（每300-500字配1张）？

**敏感词预防**：
- [ ] 外国政治人物名已替换为音译变体？
- [ ] 冲推荐文章标题/正文无"赚钱/搞钱/副业"等算法敏感词？
- [ ] 无广告法极限词（最佳/第一/最强/绝对/100%）？

**朱雀检测（最后一步——如有需要）**：
- [ ] 所有禁止模式0检出？
- [ ] 无书面语/AI句式？
- [ ] 无说教式金句/主题升华？
- [ ] 符合事件链驱动原则？

全部通过后，进入第6步易撰检测，检测通过后输出最终稿件。

**内容5要素**（来自25讲第06讲）：

## 拆解方法论

**拆解是提升写作最快的方法。**

**选标4维度**（来自25讲第13讲）：
1. **数据维度**：阅读量、点赞量远超平均的文章
2. **热度维度**：最近1-3个月的热文，时效性参考价值大
3. **学习价值**：和你定位越接近的，越值得拆
4. **补短板**：你弱什么就拆什么

**拆解4维度**：
1. **选题标题**：为什么这个选题能火？标题用了哪个技巧？
2. **脉络结构**：用的哪种框架？段落怎么推进的？
3. **爆款成因**：是什么让这篇文章被大量转发？（情绪？价值？身份认同？）
4. **内容表达**：用了什么素材？开头结尾怎么写的？金句怎么落的？

**拆解后应用5技巧**：
1. 找同定位多篇爆文，提炼共性
2. 提炼选题角度和标题套路
3. 总结框架结构和素材方法
4. 重点学爆文开头和结尾
5. 合并创建灵感库，持续积累

## 销售软文专题

**销售软文5要点**（来自25讲第23讲）：
1. **既要高阅读又要精准**：标题吸引泛流量，内容筛选精准用户
2. **创造希望，不制造焦虑**：让读者看到可能性，而不是恐惧
3. **反复磨到90分**：销售软文差10分，转化差一倍
4. **验证逻辑，避免自嗨**：让朋友读一遍，看能不能被打动
5. **暗中解决用户疑惑**：预判读者会问的问题，在文中自然回答

**销售软文4个选题方向**：
1. **讲用户渴望的结果**：先让他看到"我也能这样"
2. **讲代入感强的改变故事**：和自己越像的人的故事，越能打动
3. **讲解决方案**：直接给方法，自然过渡到"更系统的方案在课程里"
4. **讲大痛点**：这个问题不解决，后果是什么

**销售软文禁忌**：
- ❌ 一上来就卖——先给价值，再卖
- ❌ 制造焦虑逼购买——创造希望才对
- ❌ 夸大效果/虚假承诺——说真话，长期信任比短期转化重要
- ❌ 产品植入太硬——用故事/案例自然过渡

## IP长文专题

**IP长文4大价值**（来自25讲第18讲）：
1. **传播力**：真实故事最容易被转发
2. **信任**：读者了解你之后才会信任你
3. **完整画像**：一篇好IP文 > 100篇零散文章
4. **长期复用**：可以反复引用、置顶、发朋友圈

**IP长文6要素**：
1. **为什么写**：开头交代动机，真诚最重要
2. **我是谁**：背景、经历、转折
3. **重在过程讲变化**：不是展示结果，是讲怎么走到今天的
4. **用成就积累信任**：数据、案例、里程碑
5. **有价值才有动机**：你的故事对读者有什么启发？
6. **真实是根基**：编的故事经不起时间检验

**IP长文3结构**：
```
01 我是谁——背景、经历、人生关键节点
02 我在做什么——现在的状态、为什么选择这条路
03 为什么做——价值观、长期主义、对读者的承诺
```

## 商稿与干货文的区分处理

- **商稿**（含产品植入）：天然被微信推荐算法压制，不指望系统推荐。标题用数字+反常识保粉丝打开率。产品植入密度控制，避免被判定为商业推广整篇压制。
- **纯干货文**（无产品引流）：标题情绪烈度收一点，用疑问式或留白式。用干货"喂"算法拿推荐。
- **发布策略**：商稿与干货文穿插发布。用纯干货"喂"算法拿推荐，商稿吃粉丝打开率。
- **销售软文**：介于商稿和干货之间。有价值内容占70%+，产品/课程自然过渡占30%-。核心是"先帮读者，再推荐产品"。
